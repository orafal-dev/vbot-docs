title: Engine & Features

Use `Engine` for bot feature configuration and actions. Configuration is exposed through explicit getters and setters — do not assume that feature objects or their internal fields are available. Getter tables are **detached snapshots**; changing a returned table does not change the bot. Call the matching setter.

Prefer `Engine.Features` for new scripts. Older globals such as `Features`, `Inventory`, `PVPTools`, `MagicShooter`, and `Targeting` remain for compatibility.

## Feature activation

Control built-in bot features by numeric `BotFeatureId` or name string. Public IDs range from `HEALER` through `TIMER_ACTIONS`. Objects Dumper and other internal services are reserved and blocked.

| Function | Description |
| --- | --- |
| `Engine.Features.IsActive(featureIdentifier)` | Check if feature is on |
| `Engine.Features.Enable(featureIdentifier)` | Enable feature |
| `Engine.Features.Disable(featureIdentifier)` | Disable feature |
| `Engine.Features.Toggle(featureIdentifier)` | Toggle feature |
| `Engine.Features.SetActive(featureIdentifier, active)` | Set explicit state |
| `Engine.Features.GetName(featureIdentifier)` | Feature display name |
| `Engine.Features.GetAllFeatureIds()` | All feature IDs |
| `Engine.Features.GetActiveFeatures()` | Currently active features |
| `Engine.Features.EnableMultiple(list)` | Enable several at once |
| `Engine.Features.DisableMultiple(list)` | Disable several at once |
| `Engine.Features.DisableAllExcept(excludeList)` | Disable all except listed |
| `Engine.Features.PrintStatus()` | Print feature status |

```lua
if not Engine.Features.IsActive("HEALER") then
    Engine.Features.Enable("HEALER")
end
```

## Important behavior

- Entry indices are **1-based**. Most entry setters affect the active profile; call profile-selection functions first when a feature supports profiles.
- Explicit setters validate types and ranges and preserve feature side effects (timer resets, parsed-name caches, HUD refresh).
- All `GetEntries()` results and ID lists are copies — never mutate them expecting bot state to change.
- Configuration changes remain in memory until the bot's normal settings save runs.
- Internal services (object dumper, packet/event dispatcher, use-item queue, Validus networking) are not exposed through `Engine`.

## Magic Shooter entries

Inspect and configure existing Magic Shooter entries without rebuilding profiles:

| Function | Description |
| --- | --- |
| `Engine.MagicShooter.GetEntries(profile?)` | Detached entry snapshot |
| `Engine.MagicShooter.SetEntryRune(entryIndex, runeId, profile?)` | Replace with rune |
| `Engine.MagicShooter.SetEntrySpell(entryIndex, spellWords, profile?)` | Replace with spell |
| `Engine.MagicShooter.SetEntryEnabled(entryIndex, enabled, profile?)` | Toggle entry |
| `Engine.MagicShooter.SetEntryMonsterNames(entryIndex, names, profile?)` | Monster filter |
| `Engine.MagicShooter.SetEntryRange(entryIndex, range, profile?)` | Range |

`profile` defaults to the active profile (1-based index or exact name). Explicit setters also cover option, condition, mana/health/harmony thresholds, monster count, HP range, danger, PvP safety, ally shooting, delays, skill buffs, cast method, and pattern fields.

Replacing an action preserves enabled state, position, monster names, conditions, delays, and advanced settings. A recognized rune or spell also updates action type, range, and built-in pattern. Unknown custom actions cannot cross from rune to spell or spell to rune.

```lua
local AREA_RUNE_ENTRY = 4

local function selectAvalanche()
    local ok, err = Engine.MagicShooter.SetEntryRune(AREA_RUNE_ENTRY, 3161)
    if not ok then
        print("Could not select avalanche rune: " .. tostring(err))
    end
end
```

## Targeting entries

| Function | Description |
| --- | --- |
| `Engine.Targeting.GetEntries(profile?)` | Detached entry snapshot |
| `Engine.Targeting.SetEntryEnabled(entryIndex, enabled, profile?)` | Toggle entry |
| `Engine.Targeting.SetEntryMonsterName(entryIndex, name, profile?)` | Monster name |
| `Engine.Targeting.SetEntryMonstersIgnoreList(entryIndex, names, profile?)` | Ignore list |

Explicit setters also cover priority, danger, attack option, keep-distance option/range, HP range, anchoring/range, looting, diagonal movement, shootable, and reachable requirements. Ignore lists accept comma, semicolon, or newline separators.

### Scripter notes

- `Engine.Scripter.Start(name)` accepts only an exact `.lua` filename from `GetAvailableScripts()` — not source strings.
- A running script must call `Engine.Scripter.StopSelf()` to unload itself. Self-restart is unavailable; another script may call `Restart(name)`.

### Combo Bot notes

- `Engine.ComboBot.GetRoomState()` omits room passwords and transport details.
- Lua cannot connect, disconnect, or send raw Combo Bot room commands.
