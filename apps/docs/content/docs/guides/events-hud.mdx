## Event Proxies

Subscribe to game packet events with proxy objects:

```lua
local proxy = GenericTextMessageProxy:New("my_text_handler")
proxy:OnReceive(function(p, message)
    print("Message: " .. tostring(message))
end)
```

Available proxies and callback arguments after `proxy`:

| Proxy | Callback args |
| --- | --- |
| `GenericTextMessageProxy` | `message` |
| `BattleMessageProxy` | `message` |
| `LootMessageProxy` | `message` |
| `ContainerOpenProxy` | `containerIndex, containerName, containerID` |
| `ContainerCloseProxy` | `containerIndex` |
| `ContainerAddItemProxy` | `containerIndex, slot, item` |
| `ContainerUpdateItemProxy` | `containerIndex, slot, item` |
| `ContainerRemoveItemProxy` | `containerIndex, slot` |
| `StatsChangeProxy` | `eventData` |
| `SkillsChangeProxy` | `eventData` |
| `CreatureAddProxy` | `creatureId, creatureName, position` |
| `CreatureRemoveProxy` | `creatureId` |
| `DeathProxy` | _(none)_ |

Common pattern:

```lua
local p = SomeProxy:New("unique_handler_name")
p:OnReceive(function(proxy, ...)
    -- handle event
end)
```

Keep callbacks short and non-blocking. Use `Module.After` for deferred work.

## HUD Overlays

### ScreenText — fixed screen position

```lua
local hud = ScreenText:New("my_status_hud")
hud
    :SetColor({ r = 255, g = 255, b = 255, a = 255 })
    :SetFont("Segoe UI", 16)
    :SetText("Starting...")
    :Create()
    :SetScreenPosition(25, 80)

Module.Every("my_status_update", function()
    if Self.IsAvailable() then
        local hp = Self.GetHealthPercentage() or 0
        local mana = Self.GetManaPercentage() or 0
        hud:SetText("HP " .. hp .. "% | Mana " .. mana .. "%")
    end
end, 1000)
```

### Overlay layer — draw above Tibia panels

Every HUD element defaults to `HUDRenderLayer.MAP`. Pass `HUDRenderLayer.OVERLAY` to `New(...)` or call `SetRenderLayer` **before** `Create()` to draw over client panels outside the map rectangle. Changing the layer after `Create()` raises an error. Logical parents and children must share the same render layer.

```lua
local overlayTitle = ScreenText:New("overlay_title", HUDRenderLayer.OVERLAY)
    :SetText("Drawn above Tibia panels")
    :SetFont("Segoe UI", 20)
    :SetScreenPosition(700, 120)
    :SetZIndex(10)
    :Create()
```

### Fonts

`ScreenText` and `WorldText` use Tibia's HUD font by default. Prefer common Windows font-family names:

| Method | Description |
| --- | --- |
| `SetFont(family, pixelSize)` | Set family and size (1–256); works before/after `Create()` |
| `SetFontFamily(family)` / `SetFontSize(pixelSize)` | Update one property |
| `SetFont(nil, nil)` | Restore Tibia font and size |

### ScreenImage — icons with labels

```lua
local icon = ScreenImage:New("my_icon")
icon
    :SetItemId(3031)
    :SetSize(32, 32)
    :SetLabel("ON", { r = 0, g = 255, b = 0, a = 255 }, 0, -16)
    :Create()
    :SetScreenPosition(100, 100)
```

- `SetLabel(text, color, offsetX, offsetY)` — attach or update label text (also available on `WorldImage`)
- `SetSourceBase64(base64Png)` / `SetSourceBytes(pngBytes)` — embed PNG data (raw Base64, `data:image/png;base64,...`, or byte arrays); PNG-only, max 4 MiB / 2048×2048
- `SetParent(parent_id)` — parent to draggable ScreenText; children follow drag movement
- `SetZIndex(zIndex)` — draw order within the selected render layer
- `SetDragTarget(target)` / `SetOnDragEnd(callback)` — advanced drag behavior on screen elements

Choose exactly one image source setter (`SetSource`, `SetSourceBase64`, `SetSourceBytes`, `SetItemId`, or `SetItemName`); calling another replaces the prior selection.

All HUD classes support `New`, `Create`, `Remove`, `SetEnabled`, `SetZIndex`, `SetRenderLayer`, `SetParent`, and `ClearParent` as applicable. Setters return the same object for chaining. `Engine.HUD` element operations retain per-script ownership — a script cannot mutate another script's HUD elements. `Engine.HUD.UpdateFont` updates font family/size for an existing element id.

### World overlays

World-positioned elements for map debugging:

```lua
local pos = Self.GetMousePositionInWorld()
if pos then
    WorldBox:New("debug_box", pos.x, pos.y, pos.z)
        :SetSize(32, 32)
        :SetColor({ r = 255, g = 0, b = 0, a = 60 })
        :SetBorderColor({ r = 255, g = 0, b = 0, a = 255 })
        :SetLifetime(3000)
        :Create()
end
```

Available world classes: `WorldText`, `WorldBox`, `WorldImage`

Alignment constants: `HorizontalAlignment`, `VerticalAlignment`  
Render layers: `HUDRenderLayer.MAP`, `HUDRenderLayer.OVERLAY`

## Hotkeys

Register keyboard combos. **alt** is not supported for `RegisterCombo` callbacks, but `SendKey` / `SendCombo` can send alt combinations to the client:

```lua
Hotkeys.RegisterCombo({
    id = "say_hi_hotkey",
    combo = "ctrl+h",
    callback = function()
        if Self.IsAvailable() then
            Self.Say("hi")
        end
    end,
    name = "Say hi",
    trigger_on_keydown = false,
})

-- Queue synthetic input to the injected Tibia window (clientOnly defaults to true)
Hotkeys.SendKey("f1")
Hotkeys.SendCombo("ctrl+shift+f9")
Hotkeys.SendCombo("alt+f1")
Hotkeys.SendCombo("ctrl+f1", false) -- route through normal ImGui/bot/Lua handling
```

A `true` return from `SendKey` / `SendCombo` means the key sequence was queued, not that Tibia had an action assigned. Extended keys (insert, delete, arrows, etc.) default to `extended = true` in `Hotkeys.ParseCombo`.

## Sound

Play bot sounds and alerts:

```lua
Sound.Play({ sound_id = BotSoundId.LOW_HEALTH })
Sound.Play({ sound_name = "low_mana" })
Sound.Play({ file_path = "path/to/sound.wav", instant = true })
```

`Sound.Play` and `Sound.IsQueued` options must identify exactly one source: `sound_id = BotSoundId.*`, `sound_name = string`, or `file_path = string`. `Sound.Play` also accepts `instant = boolean`.

| Function | Description |
| --- | --- |
| `Sound.Play(options)` | Play sound |
| `Sound.Stop()` | Stop current |
| `Sound.ClearQueue()` | Clear queue |
| `Sound.IsPlaying()` / `IsQueued(options)` | Status |
| `Sound.GetQueueSize()` | Queue length |
| `Sound.SetMinDelay(delayMs)` | Minimum delay between queued sounds |
| `Sound.GetCurrentDuration()` / `GetFileDuration(path)` | Duration helpers |

Prefer these canonical `Sound.*` methods rather than legacy global sound helper functions.
