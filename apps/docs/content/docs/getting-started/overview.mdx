ValidusBot scripts extend the bot with custom logic — from simple hotkey handlers to full cavebot action pipelines, HTTP integrations, and persistent storage. Scripts run inside the bot's embedded Lua runtime with access to curated core wrapper libraries.

## Script location

User scripts live in the bot's user **Scripts** directory. Core helper libraries ship in a **`Scripts/core`** subdirectory and are loaded automatically when your script runs. You do not need to manually `require` core modules — they are available as global PascalCase tables.

## Public API naming

All user-facing modules use **PascalCase**:

```lua
local hp = Self.GetHealthPercentage()
local monsters = Creatures.GetVisibleMonsters(true)
local player = Creature.GetLocalPlayer()
```

Removed compatibility aliases (`self.*`, `map.*`, `engine.ammoRefill.*`, `CaveBot`, `_G.position`) must not be used in new scripts.

## What is documented here

This documentation reflects:

- The official Lua scripting specification maintained for ValidusBot
- The public surface exported by each file in `Scripts/core`

If a function is not listed in the [API Reference](/docs/api-reference), treat it as **unavailable** for user scripts. Internal helpers and undocumented low-level globals are implementation details.

## Module overview

| Module | Purpose |
| --- | --- |
| `Module` | Script scheduling (timers, repeating tasks) |
| `Self` | Local player state and actions |
| `Creature` / `Creatures` | Creature wrappers and screen scans |
| `Map` / `Position` / `Minimap` | Tile interaction and pathfinding |
| `Item` / `Container` / `Inventory` | Items, backpacks, equipment |
| `Spells` / `Cooldowns` | Spell and item cooldown checks |
| `Cavebot` | Walker, lure, and action orchestration |
| `Engine` | Feature configuration (healer, shooter, targeting, delays, …) |
| `Features` / `Engine.Features` | Enable/disable bot features |
| `Hotkeys` | Keyboard combo registration |
| `ChatChannel` / `ChatChannelStorage` | Chat channel helpers |
| `VIP` | VIP list queries |
| `NpcTradeStorage` | NPC trade window state |
| `Json` | Native JSON encode/decode |
| `Http` / `WebSocket` | Coroutine-friendly networking |
| `Storage` | Persistent script-scoped values |
| `Event Proxies` | Packet/event callbacks |
| `ScreenText` / `WorldBox` / etc. | HUD overlays |
| `Sound` | Audio alerts |
| `Game` | Login/session and window helpers |
| `lua_consts` tables | Shared enums and constants |
